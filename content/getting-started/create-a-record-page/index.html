Now that you've learned how to get a {{ stache.config.product_name_short }} app up and running, let's build a page with some content. {{ stache.config.product_name_short }} applications feature two common page types: record pages and tabbed pages.

On record pages, information about specific records is usually displayed in two distinct sections.

Summary Section

The first type of section is the summary section. The summary resides at the top of the page and provides a general overview of a record, such as the record name, description, perhaps a profile picture, etc. To add a summary section to your page, use a div with the .bb-page-header CSS class. Tiles Section

The second type of section is the tile section. This section usually consists of two columns of tiles (or one column for extra-small screen sizes like mobile phones) that break out more specific information about the record into logical sections. Each tile has a header for the tile's title and a body for the tile's content. Tiles are collapsible, and if they used in conjunction with a tile dashboard, tiles can also be rearranged by users.

This example shows a page with a static tile layout. To define a dynamic tile layout from a configuration object (such as from user preferences) you can use the tile dashboard directive.

Tiles can display data in all kinds of formats, whether it's a list of fields and their values, a repeater view of records, a chart, or anything else you can imagine. You can find into more detail about the common data elements inside tiles in the Components section, but for now let's move on to the next kind of page.

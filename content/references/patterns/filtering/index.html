There are four levels of complexity we support for filtering. Use the approach that best fits the scenario for which you are designing.

1-3 fields, frequently needed

In cases where there are a small number of fields on which to filter and the user will frequently need to change those filters, use an inline always-visible filter.

1-3 fields, infrequently needed

In cases where there are a small number of fields on which to filter but the need to change those filters will be infrequent, use an inline user-invoked filter.

4+ fields, can be grouped together

In cases where there are a moderate number of fields on which to filter but the number is small enough that the user can easily understand them if they are on one form, use a modal single-page filter.

4+ fields, need to be grouped into separate categories

In cases where there are a large number of fields on which to filter and the usability of the form will be helped by breaking those fields up into related groups, use a modal categorized filter.
